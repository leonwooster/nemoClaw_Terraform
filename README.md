# NemoClaw Azure VM Terraform

This Terraform project provisions a cost-optimized Ubuntu 22.04 Gen2 VM on Azure for NemoClaw experiments.

## What it creates

- Resource group
- Virtual network and subnet
- Static public IP
- Network security group
- Linux VM (`Standard_B2s` by default)
- Daily Azure auto-shutdown schedule
- SSH keypair generated by Terraform

## Why this size

`Standard_B2s` is a practical low-cost starting point if you are targeting roughly RM100-RM200 per month and auto-shutdown daily. Actual spend depends on runtime hours, outbound traffic, disk, and Azure pricing in your subscription.

## Important note about auto-shutdown

The default schedule is:
- `1600 UTC`
- equivalent to `12:00 AM` Malaysia time (`UTC+8`)

## Files

- `main.tf` - core Azure resources
- `variables.tf` - input variables
- `outputs.tf` - useful outputs after apply
- `cloud-init.yaml` - base bootstrap script
- `terraform.tfvars.example` - sample values

## Prerequisites

- Terraform 1.6+
- Azure CLI logged in with permission to create networking and VM resources

## Commands

```bash
az login
az account show
```

Clone or copy this folder, then:

```bash
cp terraform.tfvars.example terraform.tfvars
terraform init
terraform plan -out tfplan
terraform apply tfplan
```

## After apply

Get outputs:

```bash
terraform output
terraform output public_ip_address
terraform output ssh_command
```

SSH into the VM:

```bash
ssh -i ./id_rsa_nemoclaw azureuser@<PUBLIC_IP>
```

## Check cloud-init logs

```bash
sudo cloud-init status --wait
sudo tail -n 200 /var/log/cloud-init-output.log
sudo tail -n 200 /var/log/bootstrap-nemoclaw.log
```

## Manual NemoClaw install on the VM

```bash
curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash
nemoclaw --help
```

## Destroy

```bash
terraform destroy
```

## Security recommendation

Do not leave SSH open to the world in a long-lived deployment. Set `ssh_source_cidr` to your own public IP in `/32` form.
